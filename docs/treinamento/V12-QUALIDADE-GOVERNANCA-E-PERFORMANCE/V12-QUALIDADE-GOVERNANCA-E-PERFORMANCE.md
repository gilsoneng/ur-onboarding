# V12 — Qualidade, governança e performance

## Produção

- Duração: 12 minutos.
- Público: toda pessoa que entrega dado, modelo ou consumo.
- Responsável pela gravação: especialista de dados/plataforma, a designar.
- Aprovação técnica: André Alves.
- Publicação: Dayana Viana.

## Objetivo observável

O participante define controles mensuráveis para qualidade, rastreabilidade, permissões e custo de uma entrega CDF.

## Preparação

Usar painel fictício de execução, contrato de dados e consulta limitada. Preparar três exemplos: dado atrasado, chave duplicada e consulta ampla.

## Roteiro cronometrado

| Tempo | Fala e tela |
|---|---|
| 0:00–1:30 | Definir qualidade como regra mensurável, não como impressão após olhar um dashboard. |
| 1:30–4:00 | Mostrar controles de entrada: completude, unicidade, tipo, atraso e rejeições. |
| 4:00–6:00 | Mostrar rastreabilidade: lote, fonte, versão de transformação e evidência de execução. |
| 6:00–8:00 | Cobrir governança: dono do dado, classificação, permissões e retenção conforme política do projeto. |
| 8:00–10:00 | Demonstrar por que filtros, seleção mínima de campos e paginação afetam desempenho e custo. |
| 10:00–12:00 | Montar uma lista curta de métricas e limites de alerta para a primeira entrega. |
