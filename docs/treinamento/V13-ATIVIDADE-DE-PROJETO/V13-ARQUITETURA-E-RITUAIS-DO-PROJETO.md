# V13 — Arquitetura e rituais do projeto

## Produção

- Duração: 12 minutos.
- Público: pessoa entrando em um projeto Horizontal + UR.
- Responsável pela gravação: Tech Lead ou responsável de projeto, a designar.
- Aprovação técnica: André Alves.
- Publicação: Dayana Viana.

## Objetivo observável

O participante localiza o contexto do projeto, entende limites entre componentes e sabe em qual ritual tomar decisões técnicas.

## Preparação

Usar o canvas de arquitetura específico do projeto já aprovado, sem nomes de sistemas restritos, ou versão sanitizada. Abrir `05-DOCUMENTACAO-DO-PROJETO.md` e agenda de rituais.

## Roteiro cronometrado

| Tempo | Fala e tela |
|---|---|
| 0:00–1:00 | Apresentar o objetivo do projeto e a decisão operacional que ele atende. |
| 1:00–3:30 | Percorrer o canvas: fontes, integrações, CDF, consumidores, suporte e limites de responsabilidade. |
| 3:30–5:30 | Mostrar o repositório, ambientes, documentação oficial e critérios para mudança. |
| 5:30–7:30 | Explicar rituais: triagem, refinamento, revisão técnica, demo e retro; indicar entradas e saídas de cada um. |
| 7:30–9:30 | Demonstrar como uma dúvida passa de backlog para decisão documentada. |
| 9:30–11:00 | Mostrar canais de escalonamento e o que não deve ser resolvido por mensagem informal. |
| 11:00–12:00 | Indicar o primeiro artefato que a pessoa deve revisar e atualizar. |

## Entrega do participante

Registrar um resumo de uma página do contexto do projeto e apontar o ritual onde cada tipo de decisão será levado.

## Checklist de gravação

- [ ] Usar apenas arquitetura aprovada/sanitizada.
- [ ] Diferenciar canal de decisão de canal de aviso.
- [ ] Identificar donos de cada fronteira técnica.
