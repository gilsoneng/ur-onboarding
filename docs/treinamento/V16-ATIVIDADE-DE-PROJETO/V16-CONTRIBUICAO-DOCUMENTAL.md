# V16 — Contribuição documental e ambientação contínua

## Produção

- Duração: 10 minutos.
- Público: toda pessoa que encerra uma atividade de onboarding ou projeto.
- Responsável pela gravação: facilitador técnico/documental, a designar.
- Aprovação técnica: André Alves.
- Publicação: Dayana Viana.

## Objetivo observável

O participante identifica documentação desatualizada, propõe uma alteração revisável e fecha a atividade com evidência que ajuda a próxima pessoa.

## Preparação

Abrir `05-DOCUMENTACAO-DO-PROJETO.md`, um exemplo sanitizado de página antes/depois e uma atividade DOC do backlog. Usar repositório ou wiki de demonstração.

## Roteiro cronometrado

| Tempo | Fala e tela |
|---|---|
| 0:00–1:00 | Explicar a regra: se uma atividade mudou a forma de operar, a documentação também precisa mudar. |
| 1:00–3:00 | Mostrar a taxonomia: contexto, arquitetura, integração, operação, decisão e ambientação. |
| 3:00–5:00 | Demonstrar como localizar uma lacuna usando o radar de documentação e relacioná-la a uma atividade. |
| 5:00–7:00 | Fazer uma alteração pequena: propósito, passo verificável, dono, data de revisão e links de evidência. |
| 7:00–8:30 | Mostrar revisão por pares/técnica e como registrar decisão sem duplicar informação. |
| 8:30–10:00 | Fechar com a definição de pronto: entrega técnica, evidência e documentação sincronizadas. |

## Entrega do participante

Abrir uma contribuição documental ou atualizar uma página existente com dono, data de revisão, procedimento verificável e link para o item de backlog.

## Checklist de gravação

- [ ] Mostrar uma mudança concreta, não apenas a estrutura ideal.
- [ ] Preservar links e decisões existentes quando não forem alterados.
- [ ] Encerrar conectando documentação, backlog e aprovação.
