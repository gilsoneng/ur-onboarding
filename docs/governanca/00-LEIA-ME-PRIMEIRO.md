# Pacote de Onboarding CDF — Horizontal + UR

Este é o ponto de entrada para qualquer pessoa que receber uma atividade da trilha. O pacote foi desenhado para que o executor não precise reconstruir o contexto por conversas paralelas.

## Como usar este pacote em 3 camadas

Esta entrega torna **evidente** o que cada pessoa precisa fazer. Leia nesta ordem:

1. **Pitch (explicar em minutos)** — `pitch/Pitch-Trilha-Onboarding-CDF.pptx` para
   apresentar a trilha; `pitch/TEMPLATE-Pitch-do-Projeto.pptx` para voce explicar o
   projeto onde vai atuar (apoio para V13 e a demo EVAL-01).
2. **Checklist (o que fazer)** — `checklists/CHECKLIST-MESTRE.md` e
   `docs/treinamento/Vxx/CHECKLIST.md`.
3. **Documentacao (aprofundar)** — pasta `docs/` (governanca, treinamento, entrada).

Comece pelo `docs/entrada/INDICE.md` para o mapa completo.

## Resultado esperado

Cada entrega deve deixar uma evidência reutilizável: vídeo publicado, prática reproduzível, atualização documental aprovada ou avaliação registrada. Nenhuma atividade termina apenas com a mensagem “feito”.

## Como usar este pacote

1. Abra o arquivo da sua atividade; não comece por um documento aleatório.
2. Confirme o **dono da validação** antes de produzir qualquer material.
3. Use somente fontes e ambientes permitidos. Nunca coloque token, senha, dado pessoal ou dado restrito do cliente no material.
4. Produza a evidência indicada e registre o link no backlog.
5. Peça revisão no canal definido pelo projeto; corrija o que for devolvido.
6. Considere a atividade concluída somente quando o critério de aceite do arquivo estiver atendido.

## Convenções do pacote

- Arquivos com prefixo numérico definem a ordem de leitura.
- `Vxx` identifica um vídeo; `ONB`, `CDF`, `DOC`, `PRJ` e `EVAL` identificam itens de backlog.
- Use títulos objetivos: `V07 - Ingestão via SDK`, não “vídeo novo”.
- Links devem apontar para a fonte de verdade, não para mensagens de chat.
- Toda alteração relevante deve informar dono, data de revisão e motivo.

## Critério geral de aceite

- A entrega pode ser localizada sem pedir ajuda a quem a criou.
- Outra pessoa sabe qual público usa o material, onde está a fonte e quem o mantém.
- A evidência não expõe informação sensível.
- Há validação registrada de quem tinha autoridade para aprovar.
